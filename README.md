# Sealants4All Storefront

Production Next.js 15 + TypeScript + Tailwind v4 + PWA implementation of the
**Sealants4All V2 storefront homepage** — a UK B2B construction-materials site
(sealants, adhesives, fixings, EWI systems). Modeled on Solseal × Screwfix ×
Toolstation conventions: trade-led, convention-first, conversion-optimized.

The original design brief is preserved in [`design-refs/`](./design-refs) (HTML
prototype + JSX/CSS references) and was the source for this build.

---

## Stack

- **Framework:** Next.js 15 (App Router), React 19, TypeScript (strict)
- **Styling:** Tailwind CSS v4 with CSS variables for design tokens
- **State:** Zustand (cart persisted to `localStorage` under `s4a-v2:cart`)
- **Forms:** react-hook-form + zod
- **Icons:** lucide-react
- **Toasts:** sonner
- **PWA:** [`@serwist/next`](https://serwist.pages.dev) with custom cache
  strategies (cache-first for images, network-first for pages, SWR for `/api/*`)

## Quick start

```bash
pnpm install
pnpm dev          # http://localhost:3000
pnpm typecheck    # strict TS
pnpm lint
pnpm build
pnpm start        # production server
```

> Note: the service worker is disabled in `next dev`. Run `pnpm build && pnpm
> start` to test PWA behaviour locally.

## Replacing the placeholder assets

This build ships **placeholder** artwork so layout works:

- Procedural striped diagonal placeholders for all product/lifestyle imagery
  (see `components/ui/Placeholder.tsx`)
- PWA icons generated by `scripts/generate-icons.mjs` (brand-navy with white
  "S4ALL" wordmark, cta-orange "4")

Replace by:

1. Drop real PNG/JPG product photography under `public/products/` and swap
   `Placeholder` usage in `components/product/ProductCard.tsx` for
   `next/image`.
2. Replace `public/icons/*.png` with a 1024×1024 master rendered out via your
   designer's preferred tool. Delete `scripts/generate-icons.mjs` once real
   icons are in.

## Where to swap in Medusa later

The data layer in `lib/data/` is intentionally framework-free. To switch to
Medusa:

1. Add `MEDUSA_BACKEND_URL` to `.env`.
2. Replace each `export const X = [...]` in `lib/data/*.ts` with a
   server-component-friendly async fetcher (e.g. `export async function
   getProducts(): Promise<Product[]>`).
3. Update consumers (homepage components) — most are already server components
   and accept the data shape as-is.

## Deployment

The app is Vercel-ready (`next start` works anywhere Node runs):

```bash
vercel
# or, for a static-ish edge deploy:
pnpm build && pnpm start
```

Set `NEXT_PUBLIC_*` vars on the host. The service worker is only generated on
production builds (`NODE_ENV=production`).
